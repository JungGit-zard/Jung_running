# Jumping Girl - Food Escape | 도감 시스템 기획안

> **목적**: 캐주얼 러닝 게임에 도감 시스템을 추가하여 게임의 재미와 수집 요소를 강화

---

## 1. 도감 시스템 아이디어 분석

### 1.1 후보 아이디어

#### A) 음식 도감 (Food Collection)
**개념**: 부순 음식들의 통계와 정보를 수집하는 도감
- **장점**: 
  - 게임의 핵심 요소(음식)와 직접 연관
  - 수집 동기 부여 (모든 음식 부수기)
  - 통계 추적으로 성장감 제공
- **단점**: 음식 종류가 5개로 제한적 (확장 가능)

#### B) 성과/업적 도감 (Achievement Collection)
**개념**: 달성한 기록과 업적을 모으는 도감
- **장점**:
  - 장기적인 목표 제공
  - 다양한 플레이 스타일 유도
- **단점**: 구현 복잡도가 높음

#### C) 통계 도감 (Statistics Collection)
**개념**: 플레이 통계를 시각화하는 도감
- **장점**: 간단한 구현
- **단점**: 수집의 재미가 부족

#### D) 복합 도감 (Hybrid Collection)
**개념**: 음식 도감 + 성과 도감 + 통계를 결합
- **장점**: 가장 풍부한 콘텐츠
- **단점**: 구현 복잡도가 높음

### 1.2 최종 선택: **음식 도감 (Food Collection)**

**선택 이유**:
1. 게임의 핵심 메커니즘과 직접 연관
2. 구현이 상대적으로 간단하면서도 재미있음
3. 향후 음식 추가 시 확장 용이
4. 캐주얼 게임에 적합한 수집 요소

## 2. 음식 도감 상세 기획

### 2.1 수집 항목

각 음식마다 다음 정보를 수집:

| 항목 | 설명 | 예시 |
|------|------|------|
| **이모지** | 음식 아이콘 | 🍎 |
| **이름** | 음식 한글명 | 사과 |
| **영문명** | 음식 영문명 | Apple |
| **부순 횟수** | 총알/폭탄으로 부순 횟수 | 127회 |
| **처음 발견** | 처음 부순 날짜/시간 | 2026-02-01 14:23 |
| **마지막 발견** | 마지막으로 부순 날짜/시간 | 2026-02-01 15:45 |
| **설명** | 음식에 대한 재미있는 설명 | "빨갛고 둥근 과일. 건강에 좋지만 여기서는 장애물이다!" |
| **희귀도** | 등장 빈도 (일반/희귀/초희귀) | 일반 |
| **수집 상태** | 발견/미발견 | ✅ 발견됨 |

## 3. UI/UX 기획안

### 3.1 접근 방법

**메인 메뉴에서 접근**:
- 타이틀 화면에 "도감" 버튼 추가
- 위치: "게임 시작" 버튼 위쪽 (y: 440)
- 크기: 200×56 (게임 시작 버튼과 동일)
- 색상: 민트녹색 (#5FD9B0) 또는 보라색 (#9B59B6)으로 구분

### 3.2 도감 화면 레이아웃

```
┌─────────────────────────┐
│  [← 뒤로]    도감        │  ← 상단 헤더
├─────────────────────────┤
│                         │
│  🍎  🍔  🍕  🍟  🌭    │  ← 음식 그리드 (5개)
│                         │
│  [선택된 음식 상세 정보] │  ← 하단 상세 패널
│                         │
│  이름: 사과             │
│  부순 횟수: 127회       │
│  처음 발견: 2026-02-01  │
│  설명: 빨갛고 둥근...   │
│                         │
└─────────────────────────┘
```

### 3.3 화면 구성 요소

#### A) 상단 헤더
- **뒤로 가기 버튼**: 왼쪽 상단, "←" 또는 "뒤로"
- **제목**: 중앙, "도감" 또는 "Collection"
- **닫기**: ESC 키 또는 뒤로 가기 버튼

#### B) 음식 그리드 (중앙)
- **레이아웃**: 5개 음식을 가로로 나열 또는 2×3 그리드
- **카드 형태**: 각 음식은 카드 형태로 표시
  - 크기: 80×80 (이모지 영역)
  - 배경: 반투명 흰색 또는 어두운 배경
  - 테두리: 발견된 음식은 밝은 테두리, 미발견은 회색
- **상태 표시**:
  - ✅ 발견됨: 밝게 표시, 클릭 가능
  - ❓ 미발견: 어둡게 표시 (회색 오버레이), 클릭 불가
- **호버 효과**: 마우스 오버 시 약간 확대 (1.1배)

#### C) 하단 상세 패널
- **위치**: 화면 하단 1/3 영역
- **내용**:
  - 음식 이모지 (큰 크기, 64px)
  - 이름 (한글 + 영문)
  - 부순 횟수 (큰 숫자로 강조)
  - 처음 발견 날짜
  - 마지막 발견 날짜
  - 설명 텍스트
- **스타일**: 
  - 배경: 반투명 검은색 (rgba(0,0,0,0.7))
  - 텍스트: 흰색, 나눔명조 폰트
  - 레이아웃: 왼쪽 정렬, 간격 있게

### 3.4 인터랙션

#### 마우스/터치
- **음식 카드 클릭**: 해당 음식 상세 정보 표시
- **뒤로 가기 버튼**: 타이틀 화면으로 복귀
- **ESC 키**: 타이틀 화면으로 복귀

#### 키보드
- **←/→ 화살표**: 음식 카드 간 이동 (선택)
- **Enter/Space**: 선택된 음식 상세 정보 표시
- **ESC**: 타이틀 화면으로 복귀

### 3.5 시각적 피드백

- **카드 호버**: 약간 확대 (scale 1.1), 밝기 증가
- **카드 선택**: 테두리 강조 (현광라임 녹색)
- **애니메이션**: 카드 전환 시 페이드 효과
- **미발견 음식**: 회색 오버레이 + "?" 아이콘

## 4. 데이터 저장 구조

### 4.1 LocalStorage 구조

```javascript
{
  "collection": {
    "🍎": {
      "name": "사과",
      "nameEn": "Apple",
      "count": 127,
      "firstFound": "2026-02-01T14:23:00Z",
      "lastFound": "2026-02-01T15:45:00Z",
      "discovered": true
    },
    "🍔": {
      "name": "햄버거",
      "nameEn": "Burger",
      "count": 89,
      "firstFound": "2026-02-01T14:25:00Z",
      "lastFound": "2026-02-01T15:30:00Z",
      "discovered": true
    },
    // ... 나머지 음식들
  },
  "stats": {
    "totalDestroyed": 500,
    "totalPlayTime": 3600, // 초 단위
    "bestScore": 1200
  }
}
```

### 4.2 데이터 업데이트 시점

- **음식 부수기**: 총알/폭탄으로 음식을 부술 때마다 카운트 증가
- **처음 발견**: `discovered: false` → `true`로 변경 시 `firstFound` 기록
- **마지막 발견**: 매번 `lastFound` 업데이트

## 9. 예상 효과

### 게임플레이 향상
1. **수집 동기**: 모든 음식을 발견하려는 동기 부여
2. **재방문 동기**: 통계를 확인하고 싶어서 다시 플레이
3. **성장감**: 부순 횟수가 쌓이는 것을 보며 성취감

### 게임 지속성 향상
1. **장기 목표**: 모든 음식 수집 완료
2. **통계 추적**: 자신의 플레이 스타일 파악
3. **확장 가능성**: 향후 음식 추가 시 자연스러운 확장
